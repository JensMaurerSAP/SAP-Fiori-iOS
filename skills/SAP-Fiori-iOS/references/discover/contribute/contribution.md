# Contribution — How to Contribute

> SAP Fiori for iOS | Discover / Contribute

## Who Can Contribute?

Everyone within SAP, regardless of location or business unit.

**Recommendation:** Always contribute UI design together with code. Cross-platform contributions (iOS + Android) increase efficiency and feature parity.

---

## Contribution Types

| Type | Description |
|------|-------------|
| **New Component** | Add a new community component to the project |
| **Component Extension** | Extend an existing common component by reusing it |

**Before starting:** Align with the teams responsible for the mobile design system and SAP BTP SDK for iOS via the [communication channels](support.md).

---

## How to Contribute UI Design

1. **Join** the Community Components for Android & iOS Figma Project
2. **Create** your component on a new page in the Community Components for iOS Design Kit
3. **Publish** the Figma Library including your update to everyone at SAP
4. **Document** your changes in the Figma info panel

Optional: Add a design guideline markdown document to GitHub (see sample component).

---

## How to Contribute Code

1. **Add** the component to the iOS repository
2. **Provide** a short description of features and an API description
3. **Include** a screenshot or preview image
4. **Add** the component implementation to the InnerMobile Demo App

Optional: Create a feature request if other product teams could reuse it.

Your contribution will be reviewed by the project maintainers.

---

## Definition of Done

### UI Design
- Adheres to SAP Fiori for iOS guidelines and Apple HIG
- Covers all aspects of the Mobile Design Spec Cheat Sheet
- Added to Community Component Design Kit, tested, and documented via Figma info panel
- (Optional) Design guideline added to GitHub
- (Optional) Design specification provided

### Code
See Contribution Definition of Done on GitHub.

### Documentation
- Developer documentation in code (SAP coding guidelines)
- Design guideline as part of GitHub
