# Governance — How Governance Works

> SAP Fiori for iOS | Discover / Contribute

## Structure

The SAP Mobile Design System & SDK has two areas:

| Area | Owner | Notes |
|------|-------|-------|
| **Common Components** | Core Design & SDK Team | Maintained by Core team; reviewed before inclusion |
| **Community Components** | Contributing team | Contributor owns and maintains |

> Contributing to community components does **not** automatically guarantee publication in the common Mobile Design System and SDK. The Core Product Teams decide on inclusion.

---

## Roles

### Contributor (Product Teams)
- Anyone within SAP
- Contributes code, UI design, and documentation to Community Components
- Does **not** have access to merge directly into the core

### Core Product Team (SDK & Mobile Design System)
- Reviews community components for overall vision, scope, architecture, and roadmap alignment
- Holds primary responsibility for the Mobile Design System and SAP BTP SDK
- Governs: product scope, roadmap, backlog, prioritization, overall architecture, documentation

---

## Documentation

| Type | Location |
|------|---------|
| Developer documentation | In the code (follows SAP coding guidelines) |
| UI design documentation | Markdown documents in GitHub |

Documentation quality is as important as the code itself.
