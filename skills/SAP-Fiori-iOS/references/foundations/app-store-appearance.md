# App Store Appearance — Design Foundations

> SAP Fiori for iOS | Foundation

## What is it?

Guidelines for presenting SAP mobile apps in Apple's App Store — screenshots, text, and branding that convey the app's purpose and help users identify it.

---

## Screenshots

Minimum **5 screenshots** required. Use the SAP Figma template (provides required sizes and simplifies export).

**Do:**
- Choose high-quality screenshots that showcase the app's purpose and features
- Arrange in a compelling order that tells a story through the app's flow

**Don't:**
- Use staged screenshots that don't reflect implemented features
- Crop or cut off any part of a screenshot
- Showcase Apple-specific features (Search screen, Siri commands)

---

## Descriptive and Highlighted Text

Each screenshot needs a short, concise description of what is shown. Rules:
- Maximum **3 lines** of text
- 1–2 highlighted words/adjectives in **bold blue** to draw attention
- Text must match the specific feature shown in the screenshot

---

## Additional Languages

Provide translations for all highlighted text when publishing in non-English markets.

---

## Do's ✓ / Don'ts ✗

**Do:**
- Follow Apple's App Store guidelines
- Follow SAP Fiori for iOS app icon guidelines
- Use the SAP Figma template for correct sizes

**Don't:**
- Use screenshots that don't match the real app
- Omit translations for international markets
